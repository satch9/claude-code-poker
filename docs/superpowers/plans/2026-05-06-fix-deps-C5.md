### Task 2 : Upgrade `eslint` 8 → 9 + migration flat config

**Findings cibles** : C9.3 (eslint EOL).

**Files:**

- Modify: `package-lock.json`

- Create: `eslint.config.js`

- [ ] **Step 1 : Lire la config existante**

```bash

```

Noter : extends, parser, plugins, rules, ignorePatterns, env, parserOptions.

- [ ] **Step 2 : Upgrade ESLint et plugins compatibles**

```bash
npm install --save-dev eslint@^9 \
  @eslint/js \
  globals \
  eslint-plugin-react-hooks@^5 \
  eslint-plugin-react-refresh@^0.4.20
```

⚠️ `eslint-plugin-react-hooks@5` est compatible ESLint 9. La version 7 (latest) demande React 19 — out-of-scope C5.
⚠️ Si la version 5 n'existe pas pour `react-hooks`, vérifier la dernière compatible ESLint 9 sans React 19.

- [ ] **Step 3 : Créer `eslint.config.js`**

```javascript
// eslint.config.js
import js from "@eslint/js";
import tseslint from "@typescript-eslint/eslint-plugin";
import tsparser from "@typescript-eslint/parser";
import reactHooks from "eslint-plugin-react-hooks";
import reactRefresh from "eslint-plugin-react-refresh";
import globals from "globals";

export default [
  {
    ignores: ["dist/**", "node_modules/**", "convex/_generated/**", ".audit-tmp/**"],
  },
  js.configs.recommended,
  {
    files: ["**/*.{ts,tsx}"],
    languageOptions: {
      parser: tsparser,
      parserOptions: {
        ecmaVersion: "latest",
        sourceType: "module",
        ecmaFeatures: { jsx: true },
      },
      globals: {
        ...globals.browser,
        ...globals.node,
      },
    },
    plugins: {
      "@typescript-eslint": tseslint,
      "react-hooks": reactHooks,
      "react-refresh": reactRefresh,
    },
    rules: {
      ...tseslint.configs.recommended.rules,
      ...reactHooks.configs.recommended.rules,
      "react-refresh/only-export-components": [
        "warn",
        { allowConstantExport: true },
      ],
      // Warnings pré-existants any : conservés en warning
      "@typescript-eslint/no-explicit-any": "warn",
    },
  },
];
```

⚠️ Si l'API exacte des plugins (`tseslint.configs.recommended.rules`) diffère, adapter. Lire la doc plugin si besoin.

- [ ] **Step 4 : Supprimer `.eslintrc.cjs`**

```bash

```

- [ ] **Step 5 : Mettre à jour `package.json` script `lint` si nécessaire**

ESLint 9 détecte automatiquement `eslint.config.js`. Le script actuel `eslint . --ext ts,tsx --report-unused-disable-directives` devrait marcher mais `--ext` est déprécié en flat config (la sélection de fichiers passe par la config). Adapter :

- [ ] **Step 7 : Validation typecheck/test**
